# 星创想英语跟读 / 背诵作业小程序开发文档

## 1. 项目定位

本项目是一个面向老师、学生、家长的英语跟读 / 背诵作业系统。

核心流程：

老师在后台创建班级、学生、作业和任务内容；学生在微信小程序中查看作业、完成跟读或背诵录音；系统调用语音评测服务返回分数；老师查看完成率、分数、录音和学生表现；家长可以查看孩子的练习结果。

第一版只做 MVP，不做支付、商城、复杂排行榜、多机构 SaaS。

## 6. 语音评分业务字段

系统内部统一评分结构如下：

```ts
interface EvaluationResult {
  score: number
  accuracyScore: number
  fluencyScore: number
  integrityScore: number
  wordErrors: WordError[]
  comment: string
  rawResult?: any
}

interface WordError {
  word: string
  type: 'pronunciation' | 'missing' | 'extra' | 'fluency' | 'unknown'
  suggestion?: string
  score?: number
}
```

### 评分字段说明

| 字段           | 说明               |
| -------------- | ------------------ |
| score          | 总分，0-100        |
| accuracyScore  | 发音准确度，0-100  |
| fluencyScore   | 流利度，0-100      |
| integrityScore | 完整度，0-100      |
| wordErrors     | 错词或问题词       |
| comment        | 系统生成的简短反馈 |
| rawResult      | 第三方评测原始结果 |

### 分数反馈规则

| 分数   | 反馈文案                                 |
| ------ | ---------------------------------------- |
| 90-100 | 太棒了，发音非常清楚，继续保持！         |
| 80-89  | 很不错，再注意个别单词会更好。           |
| 70-79  | 已经读出来了，可以再练一遍提升流利度。   |
| 60-69  | 先慢一点，把每个单词读完整。             |
| 0-59   | 没关系，先听一遍标准音频，再重新读一次。 |

## 9. 核心业务规则

### 9.1 学生绑定规则

1. 老师创建学生后生成 bindCode。
2. 学生小程序登录后输入 bindCode。
3. bindCode 正确则绑定 student.userId。
4. 已绑定其他 userId 的学生，第一版不允许重复绑定。

------

### 9.2 作业展示规则

学生端只展示：

1. 自己所在班级的作业。
2. status = published 的作业。
3. 按 deadline 或 createdAt 倒序展示。

------

### 9.3 提交规则

1. 学生可以逐条录音评分。
2. 每条内容评分后生成或更新 submissionItem。
3. 学生完成所有必做内容后，可以提交整个作业。
4. 提交后 submission.status = submitted。
5. 提交后计算 averageScore。
6. 超过 deadline 后，第一版不允许提交。
7. 如果作业 status = closed，不允许提交。

------

### 9.4 评分规则

1. 如果 SPEECH_EVAL_MODE=mock，返回模拟评分。
2. 如果 SPEECH_EVAL_MODE=tencent，调用真实腾讯云语音评测。
3. 评分失败时 submissionItem.status = failed。
4. 学生可以重新录音覆盖失败记录。
5. 第一版保留最后一次评分结果即可。

------

### 9.5 老师查看权限

1. 老师只能查看自己班级的作业。
2. 老师只能查看自己班级学生的提交。
3. admin 可以查看全部。

## 13. 统一接口响应格式

所有接口统一返回：

```ts
interface ApiResponse<T> {
  code: number
  message: string
  data: T | null
}
```

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "code": 40001,
  "message": "未登录",
  "data": null
}
```

## 14. 命名规范
